# Running Tests with Keploy

With test cases recorded, we can now replay them using a single command.
Keploy will restart the application, replay each recorded request, and compare
the responses against what was captured.

---

## Run the Tests

```bash
keploy test -c "docker run -p 3001:3001 \
  --network keploy-network \
  --name ginRedisApp \
  gin-app:1.0"
```

<img
  src="/images/keploy_test2.png"
  alt="Keploy test execution starting"
  className="mx-auto my-4 rounded-lg shadow-md"
/>

Keploy will:
- Start the application inside a container
- Replay all requests from both `test-set-0` and `test-set-1`
- Use recorded mocks for Redis interactions
- Compare live responses against recorded ones

<Callout type="info">
  Keploy replays both test sets automatically. In this run, that means
  **5 total test cases** across two recording sessions.
</Callout>

---

## What Happens During Execution

For each test case, Keploy:

1. Sends the recorded HTTP request to the running app
2. Intercepts outgoing Redis calls and substitutes recorded mocks
3. Captures the live response
4. Compares it against the recorded response field by field

If anything differs (status code, body, headers), the test fails.

## Key Observation

4 out of 5 tests failed and that's not a sign something is broken.

This is Keploy behaving exactly as designed. The failures point to something
specific about how this application works, something worth understanding before
dismissing the results.
