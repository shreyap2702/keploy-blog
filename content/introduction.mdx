# Testing Go APIs Without Writing Tests

Writing API tests is tedious. You mock dependencies, hardcode request payloads, and maintain
test suites that drift from how your API actually behaves in production.

[Keploy](https://keploy.io) takes a different approach, it captures real API traffic and
replays it as test cases, automatically. No manual test writing. No stale mocks.

<Callout type="info">
  This tutorial uses a **Go (Gin) + Redis** application running in Docker. You'll hit real
  endpoints, watch Keploy capture them, and replay them as regression tests.
</Callout>

## What You'll Learn

- How Keploy fits into a local Docker-based dev workflow
- How to record API calls as test cases using `keploy record`
- How to replay them using `keploy test`
- Why dynamic values (like OTPs and tokens) cause test failures and what that tells you

## Why It Matters

Most API bugs are regressions something that worked before quietly breaks. Keploy catches
these by locking in real request/response pairs as your test baseline, so you're always testing
against actual behavior, not assumptions.
