# Fix: Updating the Redis Host

The issue was a single line in the Redis configuration. The app was pointing to
`localhost:6379`, which inside a container, refers to itself, not the Redis service.

---

## The Change

Open `helpers/redis/redisConnect.go` and update the `Addr` field:

```go
// Before
Addr: "localhost:6379"

// After
Addr: "redis:6379"
```

  src="/images/fix.png"

  className="mx-auto my-4 rounded-lg shadow-md"
/>

That's it. One word changed.

---

## Why This Works

Docker provides an internal DNS system for all containers on the same network.
Every container is reachable by its name no IPs, no config files needed.

Since the Redis container was started with `--name redis`, Docker automatically
resolves `redis` to that container's internal IP on `keploy-network`.

```
┌─────────────────────────┐     ┌─────────────────────────┐
│   ginRedisApp container │     │     redis container      │
│                         │     │                          │
│  app → redis:6379 ──────┼──✓  │  Redis running here      │
│                         │     │                          │
└─────────────────────────┘     └─────────────────────────┘
        Both on keploy-network — now resolving correctly
```
---

## Rebuild the Image

Since the source code changed, rebuild the Docker image:

```bash
docker build -t gin-app:1.0 .
```

---

## Verify the Fix

Run the app again via Keploy:

```bash
keploy record -c "docker run -p 3001:3001 \
  --network keploy-network \
  --name ginRedisApp \
  gin-app:1.0" \
  --container-name ginRedisApp
```
