# Setup

In this section, we prepare the application environment by building the Docker image,
creating a network, and starting Redis before running the app.

---

## Project Structure

After cloning, navigate into the sample:

```bash
cd samples-go/gin-redis
```

Here's what the `gin-redis` directory contains:

| File / Folder | Purpose |
|---|---|
| `main.go` | Entry point of the Gin application |
| `Dockerfile` | Used to build the app container |
| `docker-compose.yml` | Optional orchestration config |
| `keploy.yml` | Keploy configuration |
| `routes/` | API route definitions |
| `controllers/` | Request handler logic |
| `services/` | Business logic layer |
| `middlewares/` | Middleware functions |
| `config/` | App configuration |
| `helpers/` | Utility helpers |

## Build the Docker Image

Build a containerized version of the application:

```bash
docker build -t gin-app:1.0 .
```

This packages the Go application and its dependencies into a Docker image
tagged `gin-app:1.0`.

<Callout type="info">
  This step may take a minute on first run, Docker pulls the base Go image and
  compiles the application inside the container.
</Callout>

## Create a Docker Network

Create a dedicated network so the app container and Redis can communicate:

```bash
docker network create keploy-network
```

Without a shared network, containers are isolated by default and the app won't
be able to reach Redis by hostname.

---

## Start Redis

Run a Redis container on the same network:

```bash
docker run -p 6379:6379 --name redis --network keploy-network -d redis
```

| Flag | What it does |
|---|---|
| `-p 6379:6379` | Exposes Redis on localhost |
| `--name redis` | Names the container `redis` (used as hostname) |
| `--network keploy-network` | Attaches to the shared network |
| `-d` | Runs in detached (background) mode |

---

## Verify

Confirm Redis is running:

```bash
docker ps
```

You should see the `redis` container listed with status `Up`.

<Callout type="warning">
  If Redis isn't running, the application will fail to connect. We'll see exactly
  what that looks like in the next section.
</Callout>
